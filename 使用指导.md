# Git Monitor MCP Server — 使用指导

## 前置准备

### 1. 启动 MCP Server

在 Claude Code 中配置此 MCP Server 后，以下工具自动可用。无需手动启动服务。

### 2. 添加要监控的仓库

使用 `monitor_repo` 工具将仓库加入监控配置（也可直接编辑 `monitors.conf.json`）：

```
# 添加 GitHub 前端仓库
monitor_repo(
  name="user-frontend",
  url="git@github.com:org/user-frontend.git",
  branch="master",
  platform="github",
  repoType="frontend",
  module="用户中心"
)

# 添加 GitHub 后端仓库
monitor_repo(
  name="user-backend",
  url="git@github.com:org/user-backend.git",
  branch="master",
  platform="github",
  repoType="backend",
  module="用户中心"
)

# 添加另一个模块的后端仓库
monitor_repo(
  name="order-backend",
  url="git@github.com:org/order-backend.git",
  branch="master",
  platform="github",
  repoType="backend",
  module="订单系统"
)

# 添加内网 GitLab 仓库（Generic 平台 + Token 认证）
monitor_repo(
  name="internal-backend",
  url="git@gitlab.internal:team/backend.git",
  branch="main",
  platform="generic",
  repoType="backend",
  module="内部服务",
  apiBase="https://gitlab.internal/api/v4",
  apiTemplate="/projects/{owner}%2F{repo}/repository/commits?ref_name={branch}",
  authType="token",
  token="glpat-xxxxxxxxxxxx"
)
```

**关键参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 仓库别名，全局唯一 |
| `url` | ✅ | Git 远程 URL |
| `repoType` | ✅ | `"frontend"`（前端）/ `"backend"`（后端），决定克隆存储路径 |
| `module` | ✅ | 业务模块名（如 `"用户中心"`、`"订单系统"`），用于按模块批量操作 |
| `branch` | 否 | 监控的分支，默认 `main` |
| `platform` | 否 | 平台类型，默认根据 URL 自动检测 |

## 工具一：clone_repo — 一键代码克隆

### 存储路径规则

`clone_repo` 读取仓库配置中的 `repoType` 字段，自动决定存储位置：

```
Repository/
├── Frontend repository/      ← repoType="frontend"
│   └── {repo-name}/
│       ├── {branch}/         ← 全量克隆的代码
│       ├── 1001/             ← MR #1001 的代码
│       ├── 1002/             ← MR #1002 的代码
│       └── ...
└── Backend repository/       ← repoType="backend"
    └── {repo-name}/
        ├── {branch}/
        ├── 5001/
        └── ...
```

### 模式一：单仓全量克隆

将指定仓库的完整代码克隆到本地。

```
clone_repo(name="user-backend", mode="full")
```

执行效果：
- 目标路径：`Repository/Backend repository/user-backend/master/`
- 执行 `git clone --branch master --single-branch` 单分支克隆
- 目录已存在时提示，可加 `force=true` 强制覆盖

```
# 强制覆盖已有目录
clone_repo(name="user-backend", mode="full", force=true)
```

### 模式二：单仓增量克隆 — 按日期

将指定日期之后合入的所有 MR 逐一克隆到本地，每个 MR 一个子目录。

```
# 拉取 2026 年 6 月 13 日之后合入的所有 MR
clone_repo(
  name="user-backend",
  mode="incremental",
  sinceDate="2026-06-13"
)
```

### 模式三：单仓增量克隆 — 按 MR ID

以指定 MR 为基线，拉取该 MR 之后合入的所有新 MR。

```
# 拉取 MR #5001 之后合入的所有 MR
clone_repo(
  name="user-backend",
  mode="incremental",
  sinceMrId="5001"
)
```

### 模块批量克隆

使用 `module` 参数代替 `name`，一键克隆整个模块下的所有仓库。

```
# 全量克隆"用户中心"模块的所有仓库
clone_repo(module="用户中心", mode="full")
```

执行效果：
- 自动筛选该模块下的所有仓库（可能同时有前端和后端）
- 按各自的 `repoType` 存到对应目录（`Frontend repository/` 或 `Backend repository/`）
- 串行克隆每个仓库，最终汇总：共 N 个仓库，全部通过 / 部分失败

```
# 增量克隆"用户中心"模块下所有仓库的近期 MR
clone_repo(
  module="用户中心",
  mode="incremental",
  sinceDate="2026-06-13"
)

# 模块 + sinceMrId 增量克隆
clone_repo(
  module="用户中心",
  mode="incremental",
  sinceMrId="5001"
)
```

### 错误处理

增量克隆过程中：
- 单个 MR 克隆失败**不阻塞后续 MR**，继续处理下一个
- 模块模式下单个仓库克隆失败**不阻塞后续仓库**
- 最终汇总成功/跳过/失败数量，失败的详细信息单独列出
- 克隆失败的目录会自动清理

## 工具二：check_commits — 定时检查新提交

配合 Claude Code 的 CronCreate 实现定时轮询：

```
# 检查所有仓库
check_commits()

# 检查指定仓库
check_commits(name="user-backend")

# 检查整个模块
check_commits(module="用户中心")
```

Cron 示例：
```
/cron "*/15 * * * *" "用 git-monitor 检查所有仓库: check_commits()"
/cron "*/15 * * * *" "检查用户中心模块: check_commits(module='用户中心')"
```

## 工具三：get_commits — 即时查询提交历史

```
# 查看最近 20 条提交
get_commits(name="user-backend", count=20)
```

## 工具四：reset_watermark — 敏捷迭代水位重置

```
# 迭代启动：将水位定位到 6 月 1 日后第一个 MR 的 base commit
reset_watermark(
  name="user-backend",
  label="Sprint 26 kickoff",
  sinceDate="2026-06-01"
)

# 批量重置整个模块的水位
reset_watermark(
  module="用户中心",
  label="Sprint 26 kickoff",
  sinceDate="2026-06-01"
)
```

list_monitors()

## 典型工作流

```
# ── 第 1 步：配置仓库（按模块归类） ──
monitor_repo(
  name="user-frontend",
  url="git@github.com:team/user-frontend.git",
  branch="master",
  repoType="frontend",
  module="用户中心"
)

monitor_repo(
  name="user-backend",
  url="git@github.com:team/user-backend.git",
  branch="master",
  repoType="backend",
  module="用户中心"
)

monitor_repo(
  name="order-backend",
  url="git@github.com:team/order-backend.git",
  branch="master",
  repoType="backend",
  module="订单系统"
)

# ── 第 2 步：按模块一键全量克隆 ──
clone_repo(module="用户中心", mode="full")
clone_repo(module="订单系统", mode="full")

# ── 第 3 步：迭代启动，按模块重置水位 ──
reset_watermark(
  module="用户中心",
  label="Sprint 26 kickoff",
  sinceDate="2026-06-13"
)

reset_watermark(
  module="订单系统",
  label="Sprint 26 kickoff",
  sinceDate="2026-06-13"
)

# ── 第 4 步：日常按模块增量克隆 MR ──
clone_repo(
  module="用户中心",
  mode="incremental",
  sinceDate="2026-06-13"
)

# ── 第 5 步：定时监控（按模块或全量） ──
check_commits(module="用户中心")
check_commits()
```
